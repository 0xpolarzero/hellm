# Hellm Test Matrix

This matrix maps every `HellmFeature` member from `docs/features.ts` to an owning package, test layer, anchor test files, and evidence source.

It is intentionally the auditable checklist for product coverage. Completeness against `docs/features.ts` is enforced by `packages/orchestrator/test/feature-matrix.test.ts`.

Test seam rules are non-negotiable:

- TUI startup proof must use the real pi-hosted shell path.
- Headless and process tests must stay deterministic by replacing the external pi/model boundary with fake bridges or fake SDK modules.
- Ambient machine-dependent execution is not acceptable evidence in the default suite.
- If a test depends on a real external runtime or real credentials, it must be explicitly gated and labeled as such.
