## Design Context

### Users
Solo power-user developers using `hellm` as a desktop coding agent while working inside real repositories. They need speed, legibility, and confidence while inspecting code, sending prompts, reviewing output, and managing provider/runtime state without losing focus.

### Brand Personality
Sleek, practical, confident.

The interface should feel capable and intentional rather than playful or chatty. It should communicate sharp control, operational clarity, and a high-quality desktop-tool sensibility.

### Aesthetic Direction
Build a dual-theme desktop workbench with an industrial-editorial feel: crisp structure, strong typographic hierarchy, decisive spacing, and a neon orange activation color used with restraint.

Reference:
- `factory.ai`'s new app as a benchmark for quality and focus, not as a template to copy

Anti-references:
- generic AI SaaS card stacks
- purple/blue gradient slop
- glassy component-library aesthetics
- soft, weak, over-rounded interfaces

Motion should stay subtle and purposeful, with reduced-motion support respected.

### Design Principles
1. Make the working surface feel like a focused instrument panel, not a marketing page or chat toy.
2. Use hierarchy through typography, rhythm, and contrast before decorative effects.
3. Keep the neon orange accent rare and meaningful so it signals action, status, and attention.
4. Desktop and mobile should feel intentionally re-composed for their context, not merely scaled versions of the same layout.
5. Preserve operational trust: model, auth, transcript, artifacts, and settings should feel like one system and must not compromise the pi-owned runtime boundary.
